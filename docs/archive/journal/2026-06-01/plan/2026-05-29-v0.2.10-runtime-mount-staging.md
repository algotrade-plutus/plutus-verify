# v0.2.10 Implementation Plan — per-step staging dir closes runtime mount leak

> **For engineers executing this:** Implement task-by-task in order.
> Each task's checkboxes are bite-sized (~2-5 min). Run tests after every
> implementation step.

**Goal.** Each `plutus check` step runs in an isolated staging copy of the
repo (built by filtering cwd through `.dockerignore`), not against a live
volume mount of the maintainer's working tree. Closes the verification-
correctness gap left open by v0.2.9: host `.env` and stale host
`data/cache/` can no longer reach the container at runtime.

**Architecture.** Add `plutus_verify/spec/runtime/staging.py` with two
helpers: `populate_staging(cwd, step)` copies cwd → tmpdir respecting
`.dockerignore` (and `step.inputs` if non-empty); `extract_outputs(
staging, cwd, step)` copies the writable result paths back. The
orchestrator's `_run_one_step` wraps the existing `runner.run(...)` call
with these two helpers and changes `cwd=` from `repo_path` to the
staging path. The runner itself stays duck-typed and manifest-unaware.

**Tech Stack.** Python 3.11+, `pathspec` (new dep) for Docker-compatible
ignore-pattern matching, `tempfile.TemporaryDirectory` for staging
lifetime management, stdlib `shutil` for copies.

**Reference for context.** Feedback that motivated this plan lives at
[`~/algotrade-research/cs408-2026/Group09-BuyHighSellLow/.plutus/skill-feedback.md`](../../../cs408-2026/Group09-BuyHighSellLow/.plutus/skill-feedback.md). The
mount-bypass walkthrough is in `CHANGELOG.md [0.2.9] § Known
limitation` and `skills/plutus-transform/references/known-gotchas.md
§ G11`.

## Background: what changes for the engineer

Today (`v0.2.9`):

```
docker run --rm -v {repo_path}:/srv/repo plutus-v2:<tag> bash -lc "<cmd>"
```

After (`v0.2.10`):

```
1. staging = /tmp/plutus-stage-<step>-<rand>/
2. populate_staging(repo_path, step) → copies repo_path to staging,
   respecting .dockerignore (and step.inputs if declared)
3. docker run --rm -v {staging}:/srv/repo plutus-v2:<tag> bash -lc "<cmd>"
4. extract_outputs(staging, repo_path, step) → copies back:
   - always: .plutus/run/<step_id>/  (framework bookkeeping)
   - manifest-declared: every path matching step.outputs
5. rm -rf staging  (TemporaryDirectory handles this)
```

**Compatibility:** existing manifests with empty `step.inputs` and
empty `step.outputs` continue to work — populate copies everything
not in `.dockerignore`, and extract copies back only the framework's
own `.plutus/run/<step>/` (the manifest didn't ask for anything else).
Authors who relied on side-effects landing in cwd outside declared
`outputs:` will lose them — that's the intended discipline; surface
the loss in a follow-up release with a "files left in staging" warning.

```python
"""Per-step staging for the v2 runtime (v0.2.10+).

Closes the runtime-mount leak surfaced by Group09-BuyHighSellLow: instead
of mounting the maintainer's cwd onto the container, each step runs against
a filtered copy that respects `.dockerignore` (and, if declared,
`step.inputs`). Step outputs flow back to cwd via :func:`extract_outputs`.

This module is intentionally orchestrator-callable and runner-unaware —
the existing duck-typed `Runner.run(image=, command=, cwd=, ...)` contract
stays as-is; the orchestrator just passes the staging path as `cwd`.
"""
from __future__ import annotations

import shutil
from pathlib import Path

from plutus_verify.spec.manifest import Step


def populate_staging(cwd: Path, staging: Path, step: Step) -> None:
    """Copy `cwd` into `staging`, filtered by .dockerignore semantics."""
    for src in cwd.rglob("*"):
        if src.is_dir():
            continue
        rel = src.relative_to(cwd)
        dest = staging / rel
        dest.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src, dest)
```

```python
import pathspec
# (add this import at the top of the file)


def _load_ignore_spec(cwd: Path) -> pathspec.PathSpec:
    dockerignore = cwd / ".dockerignore"
    if not dockerignore.exists():
        return pathspec.PathSpec.from_lines("gitwildmatch", [])
    lines = dockerignore.read_text().splitlines()
    return pathspec.PathSpec.from_lines("gitwildmatch", lines)


def populate_staging(cwd: Path, staging: Path, step: Step) -> None:
    spec = _load_ignore_spec(cwd)
    for src in cwd.rglob("*"):
        if src.is_dir():
            continue
        rel = src.relative_to(cwd)
        # pathspec matches with POSIX-style paths
        rel_posix = rel.as_posix()
        if spec.match_file(rel_posix):
            continue
        dest = staging / rel
        dest.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src, dest)
```

```python
def populate_staging(cwd: Path, staging: Path, step: Step) -> None:
    spec = _load_ignore_spec(cwd)
```

```python
def extract_outputs(staging: Path, cwd: Path, step: Step) -> None:
    """Copy framework bookkeeping + manifest-declared outputs back to cwd.

    The orchestrator reads `.plutus/run/<step_id>/{stdout,stderr,meta.json}`
    to determine step outcome — that directory must always come back. Any
    additional paths the maintainer declared via `step.outputs` are also
    copied back. Anything else the script wrote inside staging is dropped.
    """
    run_dir = staging / ".plutus" / "run" / step.id
    if run_dir.exists():
        dest = cwd / ".plutus" / "run" / step.id
        dest.parent.mkdir(parents=True, exist_ok=True)
        if dest.exists():
            shutil.rmtree(dest)
        shutil.copytree(run_dir, dest)
```

```python
def extract_outputs(staging: Path, cwd: Path, step: Step) -> None:
    # Always: framework bookkeeping
    run_dir = staging / ".plutus" / "run" / step.id
    if run_dir.exists():
        dest = cwd / ".plutus" / "run" / step.id
        dest.parent.mkdir(parents=True, exist_ok=True)
        if dest.exists():
            shutil.rmtree(dest)
        shutil.copytree(run_dir, dest)

    # Manifest-declared outputs
    if not step.outputs:
        return
    outputs_spec = pathspec.PathSpec.from_lines("gitwildmatch", list(step.outputs))
    for src in staging.rglob("*"):
        if src.is_dir():
            continue
        rel = src.relative_to(staging)
        if rel.parts[:3] == (".plutus", "run", step.id):
            continue  # already handled
        if not outputs_spec.match_file(rel.as_posix()):
            continue
        dest = cwd / rel
        dest.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(src, dest)
```

```markdown
## [0.2.10] — 2026-05-29

Closes the runtime-mount verification-correctness gap that 0.2.9 left
open. Each `plutus check` step now runs in an isolated staging copy of
the repo (filtered by `.dockerignore`), not against a live volume mount
of the maintainer's cwd. Host `.env` and stale `data/cache/` files can
no longer reach the container at runtime.

### Changed — per-step staging dir replaces direct cwd mount

Before (0.2.9 and earlier):

    docker run --rm -v {cwd}:/srv/repo plutus-v2:<tag> bash -lc "<cmd>"

The mount overlaid the host's full working tree onto the container
verbatim — `.dockerignore` (a build-context concept) didn't apply.

After (0.2.10):

    1. populate_staging(cwd, staging, step) → copies cwd into a tempdir,
       respecting .dockerignore (and step.inputs, if declared)
    2. docker run --rm -v {staging}:/srv/repo plutus-v2:<tag> bash -lc "<cmd>"
    3. extract_outputs(staging, cwd, step) → copies back .plutus/run/<step>/
       (always) and any path matching step.outputs (if declared)

Inter-step state still flows via cwd: step N writes outputs back to
cwd, step N+1's staging dir picks them up at populate time. The mount
is no longer the shared-state mechanism; cwd is.

### Implication for Tier 3 bridge steps

- Host `.env` is invisible to the container — `--secrets-from-env`
  routing is now actually authoritative.
- Host `data/cache/*.parquet` from prior runs no longer short-circuits
  the bridge step. Tier 3 `data_collection` either connects to the DB
  for real or fails with "missing credentials" — both are correct.

### Implication for authors

- Existing manifests with empty `step.inputs` continue to work
  unchanged — populate copies everything not in `.dockerignore`.
- Existing manifests with empty `step.outputs` get only
  `.plutus/run/<step>/` back. Scripts that wrote useful files
  elsewhere will lose them. Declare those paths in `step.outputs:`.
- Declaring `step.inputs:` opts into tighter filtering — only
  paths matching those patterns reach staging. Recommended for
  new manifests; optional for existing ones.

### Migration

Most repos: none. Re-run `plutus check` and observe whether bridge
steps still pass. If `data_collection` now fails where it used to
succeed, your prior runs were short-circuiting on host state — wire
real env vars or a real DB connection and re-run. If a downstream
step fails because a file it expects in cwd isn't there, declare the
file in the producing step's `step.outputs`.

### Non-goals

- **Inputs-only mount in the runner.** Per-step staging is a copy,
  not a mount restriction. The container still sees its entire
  staging filesystem; only what staging *contains* is restricted.
- **Validate that scripts only read declared inputs.** Out of scope
  for 0.2.10; would require ptrace or LSM hooks. Authors are trusted
  to declare honestly.
- **Per-step staging cache reuse.** Each step gets a fresh staging
  dir. Repos where staging is expensive (very large) may want a
  shared base + per-step diff; deferred until profiling shows real
  cost.
```
